## 3. Decisiones técnicas

### 3.1 Framework de memoria: **LangGraph**

| Opción | Veredicto | Razón |
|---|---|---|
| LangGraph | ✅ Elegido | `MemorySaver` + `thread_id`. Estado explícito. Estándar 2026. |
| LangChain memory | ❌ | `ConversationBufferMemory` deprecado en favor de LangGraph. |
| Pydantic AI | ❌ | Mejor para tool-calling puro; menos expresivo para onboarding secuencial. |

### 3.2 Embeddings: dual strategy

| Modelo | Dims | Costo/1M tokens | Default |
|---|---|---|---|
| `text-embedding-3-small` (OpenAI) | 1536 | $0.02 | ✅ MVP default |
| `text-embedding-3-large` (OpenAI) | 3072 | $0.13 (6.5× más caro) | Sólo si recall insuficiente |
| `BAAI/bge-m3` (HF, local) | 1024 | $0 | Fallback sin API key |
| `intfloat/multilingual-e5-large` (HF) | 1024 | $0 | Alternativa gratis |

Switch vía `EMBED_PROVIDER=openai|hf` en `.env`.

### 3.3 LLM: OpenAI `gpt-4o-mini`

Default por balance costo/calidad. Switch vía `LLM_MODEL` en `.env`.

### 3.4 Datos: catálogo sintético para MVP

30 productos reales (CeraVe, The Ordinary, La Roche-Posay, COSRX, Drunk Elephant, Laneige, Kiehl's, etc.) en `app/data/catalog.py`. Precios en MXN. Plan de migración a dataset de Kaggle documentado.

### 3.5 Frontend: React + Vite + TS + Tailwind

Usuario pidió chat conversacional sencillo. Implementado en `frontend/` hermano de `app/`.

### Diagrama de flujo

```
[React (localhost:5173)]
         │  POST /api/chat {session_id, message}
         ▼
[FastAPI (localhost:8000)]
         │
         ▼
[LangGraph sales_agent]
   ┌──────────────────────┐
   │ onboarding_node       │  ← extrae perfil, pregunta datos faltantes
   │   └─ LLM structured   │
   └──────────┬────────────┘
              │ ready_to_recommend?
       ┌──────┴──────┐
       ▼             ▼
   [END]       [recommend_node]
                    │
                    ▼
              [RAG: Chroma]
                    │
                    ▼
                  [LLM]
                    │
                    ▼
                [END]
         │
         ▼
   Respuesta al React
```

### Memoria

`MemorySaver()` persiste por `thread_id = session_id`. El `session_id` se genera con `crypto.randomUUID()` en el frontend y se guarda en `localStorage`. Recarga de página → misma conversación.

## 6. Cómo arrancar (siguiente paso para el usuario)

```bash
# 1. Backend
cd app
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env         # editar con tu OPENAI_API_KEY
cd ..
uvicorn app.main:app --reload
# http://localhost:8000/docs

# 2. Frontend (en otra terminal)
cd frontend
cp .env.example .env         # opcional; por defecto apunta a localhost:8000
npm install
npm run dev
# http://localhost:5173
```

> **Nota de seguridad**: hay **dos archivos `.env`** separados.
> - `app/.env` → secretos (API keys). Nunca se comparte ni se sube a git.
> - `frontend/.env` → config pública (VITE_API_URL). Todo lo que empiece con `VITE_` se **embebe en el bundle del navegador**. Nunca poner secretos ahí.
